# Руководство по SEO оптимизации

## Что было сделано

### ✅ 1. Создан robots.txt
Файл `robots.txt` в корне проекта автоматически копируется в `dist/` при сборке.

**Что он делает:**
- Разрешает индексацию всех страниц сайта
- Блокирует служебные папки (.temp, .git, node_modules)
- Указывает путь к sitemap.xml

### ✅ 2. Добавлены SEO мета-теги
Все страницы теперь содержат полный набор мета-тегов:

**Базовые теги:**
- Description (описание страницы)
- Keywords (ключевые слова)
- Author (автор)

**Open Graph (для соцсетей):**
- Заголовок, описание, изображение
- Правильное отображение при репосте в Facebook, VK и т.д.

**Twitter Card:**
- Красивые карточки при репосте в Twitter/X

**Canonical URL:**
- Указывает поисковикам основной URL страницы

### ✅ 3. Интегрирован логотип в страницы ошибок
Страницы ошибок (404, 500 и т.д.) теперь имеют:
- Favicon (иконка во вкладке браузера)
- Apple Touch Icon (для iOS)
- Мета-тег robots: noindex (чтобы не индексировались)

**Важно:** Используются абсолютные пути из конфигурации, без относительных путей.

### ✅ 4. Автоматическая генерация sitemap.xml
При каждой сборке создается `dist/sitemap.xml` со всеми страницами сайта.

**Что включает:**
- Все HTML страницы (кроме ошибок)
- Дату последнего изменения
- Приоритет страницы (главная = 1.0, остальные = 0.5-0.8)
- Частоту обновления

## Как использовать

### Запуск сборки
```bash
node build-all-v3.js website
```

Автоматически создаются:
- `dist/robots.txt`
- `dist/sitemap.xml`
- Все HTML с мета-тегами
- Страницы ошибок с favicon

### Настройка SEO параметров

Отредактируйте `website/config.yaml`:

```yaml
site:
  name: "Название сайта"
  description: "Описание сайта для поисковиков"
  keywords: "ключевые, слова, через, запятую"
  baseUrl: "https://ваш-сайт.com"
```

### Для отдельных страниц

В `doc-config.yaml` можно указать специфичные параметры:

```yaml
pages:
  - file: "index.md"
    title: "Заголовок страницы"
    description: "Описание для этой конкретной страницы"
    keywords: "специфичные, ключевые, слова"
    image: "./assets/preview.png"  # Изображение для соцсетей
```

## Проверка результата

### 1. Проверить robots.txt
```bash
cat dist/robots.txt
```

### 2. Проверить sitemap.xml
```bash
cat dist/sitemap.xml
```

### 3. Проверить мета-теги
Откройте любую страницу в браузере → Просмотр кода → найдите `<meta>` теги в `<head>`

### 4. Проверить favicon на странице ошибки
Откройте `dist/404.html` в браузере → проверьте иконку во вкладке

## Что дальше?

### После деплоя на сервер:

1. **Google Search Console**
   - Добавьте сайт
   - Отправьте sitemap.xml
   - Проверьте индексацию

2. **Проверка Open Graph**
   - Facebook Debugger: https://developers.facebook.com/tools/debug/
   - Вставьте URL страницы → проверьте превью

3. **Яндекс.Вебмастер**
   - Добавьте сайт
   - Отправьте sitemap.xml

## Структура файлов

```
проект/
├── robots.txt                          # Правила для роботов
├── website/
│   └── config.yaml                     # SEO настройки
├── components/
│   ├── htmlGenerator.js                # Генерация мета-тегов
│   ├── errorPageGenerator.js           # Страницы ошибок с favicon
│   └── sitemapGenerator.js             # Генератор sitemap
└── dist/                               # Результат сборки
    ├── robots.txt                      # Скопирован из корня
    ├── sitemap.xml                     # Автоматически создан
    ├── index.html                      # С мета-тегами
    ├── 404.html                        # С favicon
    └── ...
```

## Вопросы?

Если что-то не работает:
1. Проверьте `website/config.yaml` - все ли поля заполнены
2. Запустите сборку заново: `node build-all-v3.js website`
3. Проверьте консоль на ошибки
4. Убедитесь что файлы созданы в `dist/`
